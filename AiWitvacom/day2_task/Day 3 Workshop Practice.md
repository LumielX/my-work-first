# Day 3 Workshop Practice

ไฟล์นี้ออกแบบเป็นแบบฝึกสำหรับ workshop วันที่ 3 เพื่อให้นักเรียนฝึก `ReAct`, `Tool Evidence`, และ `Recovery Proof` แบบทีละขั้น และเข้าใจแนวทางอย่างชัดเจนก่อนลงมือกับโปรเจกต์ของทีม

## เป้าหมายของแบบฝึก

- แยก `Thought`, `Action`, `Observation`, และ `Next Action` ออกจากกันให้ชัด  
- เลือก tool ให้ตรงคำถาม ไม่ใช้แบบสุ่ม  
- อธิบาย recovery ได้เมื่อผลลัพธ์ไม่ตรงที่คาด

## สิ่งที่นักเรียนต้องส่งเมื่อจบกิจกรรม

- ReAct Log อย่างน้อย 2 รอบ  
- Tool Evidence อย่างน้อย 2 ชิ้น  
- Recovery Proof 1 เคส

## ใช้กับโจทย์ไหนได้บ้าง

- หน้าเว็บเมนูโรงอาหารประจำวัน  
- เกม RPG แบบ text mode  
- ระบบจำลองการทำงานของหุ่นยนต์ในสิ่งแวดล้อม 3 มิติ

## Warm-Up 10 นาที

ให้นักเรียนตอบ 3 ข้อนี้ก่อนเริ่ม:

1. ตอนนี้ทีมกำลังพยายามตอบคำถามอะไร  
2. ถ้าจะทำแค่ 1 action ตอนนี้ ควรทำอะไร  
3. ถ้า action นั้นเสร็จแล้ว เราควรได้รู้อะไรใหม่

## กติกาง่าย ๆ ก่อนเริ่ม

1. Thought ต้องเป็นคำถามหรือเหตุผล ไม่ใช่คำสั่งล้วน  
2. Action ต้องเป็นสิ่งที่ทำได้จริง เช่น search, read file, run terminal  
3. Observation ต้องมาจากผลจริง ไม่ใช่สิ่งที่หวังว่าจะเจอ  
4. Next Action ต้องเปลี่ยนตาม observation

## Prompt ตั้งต้นที่แนะนำ

### ตัวเลือก A: หน้าเว็บเมนูโรงอาหาร

"ช่วยหาว่าหน้าเว็บต้นแบบของทีมมีไฟล์อยู่แล้วหรือยัง, ถ้ายังไม่ครบให้สรุปว่าควรมี section อะไรบ้าง, และบอกว่าข้อมูลไหนยังต้องให้คนยืนยันก่อนใช้จริง"

### ตัวเลือก B: เกม RPG แบบ text mode

"ช่วยตรวจว่าเกมของทีมเริ่มได้หรือยัง, มีจุดไหนที่ผู้เล่นยังเลือกคำสั่งไม่ได้, และมี branch ไหนที่ยังไม่มีตอนจบหรือทำให้ผู้เล่นสับสน"

### ตัวเลือก C: หุ่นยนต์ 3 มิติ

"ช่วยตรวจว่าตัวจำลองของทีมรันได้หรือยัง, หุ่นยนต์ขยับตามคำสั่งพื้นฐานได้หรือไม่, และมีจุดไหนที่ยังต้องให้คนช่วยตัดสินเรื่องกติกาความปลอดภัย"

## ตัวอย่างคำใบ้สำหรับ Thought ที่ดี

- ต้องรู้ก่อนว่าไฟล์หรือฉากเริ่มต้นมีอยู่จริงไหม  
- ต้องรู้ว่าปัญหาที่เห็นเกิดจากข้อมูลไม่ครบหรือ logic ยังไม่ต่อกัน  
- ต้องรู้ว่าผลที่ได้เปลี่ยนแผนของเรายังไง

## ตัวอย่างคำใบ้สำหรับ Observation ที่ดี

- พบไฟล์แล้ว แต่ยังไม่มี section ที่ต้องใช้  
- เกมรันได้ แต่ผู้เล่นยังเลือกทางไม่ได้  
- ตัวจำลองเปิดได้ แต่หุ่นยนต์ยังไม่หยุดก่อนชน

## ตัวอย่างการเขียน Tool Evidence

- `search`: ใช้หาไฟล์ `index.html` เพื่อดูว่าทีมมีหน้าเว็บตั้งต้นหรือยัง  
- `read file`: ใช้เปิดไฟล์เนื้อเรื่องเพื่อดูว่า branch ทาง B มีตอนจบหรือไม่  
- `terminal`: ใช้รันตัวจำลองเพื่อดูว่าหุ่นยนต์ขยับจริงหรือเปล่า

## แบบฝึกรอบที่ 3: Recovery Proof

ให้เลือก 1 ปัญหาที่เจอจริง แล้วตอบ 4 ข้อนี้

1. ปัญหาที่เจอคืออะไร  
2. Observation ไหนทำให้รู้ว่าปัญหาน่าจะเกิดจากอะไร  
3. ทีมลองแก้อะไร  
4. รู้ได้ยังไงว่าดีขึ้นจริง

## Mini Checkpoint สำหรับผู้สอนหรือเพื่อนในทีม

ก่อนบอกว่าจบ ให้เช็ก 5 ข้อนี้

1. Thought เป็นคำถามจริงหรือยัง  
2. Action ตรงกับคำถามหรือยัง  
3. Observation เป็นผลจริงหรือยัง  
4. Next Action เปลี่ยนตาม observation หรือยัง  
5. Recovery มีหลักฐานว่าดีขึ้นจริงหรือยัง

## ตัวอย่างคำถามสะท้อนคิดท้ายกิจกรรม

- รอบไหนที่ observation ทำให้ทีมเปลี่ยนแผนชัดที่สุด  
- tool ไหนช่วยลดความไม่แน่ใจได้มากที่สุด  
- ถ้าทำใหม่อีกครั้ง ทีมจะเริ่มจากคำถามอะไรแทน

## เกณฑ์ว่าถือว่าผ่านแบบฝึก

- มี ReAct อย่างน้อย 2 รอบที่ไม่ซ้ำกัน  
- มี Tool Evidence ที่โยงกับคำถามจริง  
- มี Recovery Proof ที่อธิบายเหตุและผลได้
